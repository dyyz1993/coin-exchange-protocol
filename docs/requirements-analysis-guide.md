## 🚀 实际执行指导 (v2.0 新增)

### 开发团队工作流程

#### 1. 任务领取流程
```bash
# 1. 开发者从 GitHub Issues 查看可用任务
# 2. 开发者创建特性分支
git checkout -b feature/fix-account-creation

# 3. 开发者编写代码和测试
# 4. 开发者提交代码
git add .
git commit -m "fix: 修复账户创建功能 (#175)"

# 5. 推送到远程
git push origin feature/fix-account-creation

# 6. 创建 Pull Request
# 7. 等待 Code Review
# 8. 开发总监合并 PR
```

#### 2. 开发总监协调流程
```bash
# 1. 检查 GitHub Issues
# 2. 使用 assign_task 分配任务
assign_task(to="developer-id", task="修复账户创建功能", issueNumber=175)

# 3. 定期检查团队状态
check_messages()

# 4. 审查并合并 PR
merge_pr(prNumber=123, commitMessage="合并账户创建修复")
```

#### 3. 每日站会检查清单
- [ ] 昨天完成了什么？
- [ ] 今天计划做什么？
- [ ] 遇到什么阻塞？
- [ ] 是否需要帮助？

#### 红色警报 🚨
