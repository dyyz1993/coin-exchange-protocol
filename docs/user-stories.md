```
作为 <角色>

```

**文档状态**: ✅ 已完成  
**满足验收标准**:
- ✅ 覆盖7个核心模块（账户管理、余额管理、交易、空投、任务、通知、安全）
- ✅ 每个故事有明确的验收标准
- ✅ 包含3种用户角色视角（新用户、活跃用户、管理员）

## 👥 用户角色

### 主要角色

1. **新用户** (New User)
   - 刚注册的用户
   - 需要创建账户并了解系统
   - 目标：快速开始使用系统

2. **活跃用户** (Active User)
   - 经常使用系统的用户
   - 进行交易、完成任务
   - 目标：高效管理资产

3. **管理员** (Admin)
   - 系统管理员
   - 管理用户、空投、任务
   - 目标：维护系统正常运行

## 📝 核心用户故事

### Epic 1: 账户管理

#### US-1.1: 创建账户 ⚠️ [P0 - 已知问题]
```
作为 新用户
我希望 能够快速创建账户并设置初始余额
以便 开始使用金币交易系统
```

**验收标准**:
- ✅ 可以使用唯一标识符创建账户
- ✅ 可以设置初始余额（可选）
- ✅ 账户创建后立即生效
- ✅ 重复创建被拒绝并给出清晰错误
- ✅ 无效输入得到验证提示

**当前状态**: 🔴 **严重问题** - 方法签名不匹配，参数未生效  
**相关 Issues**: #175, #171, #142, #163

**用户旅程**:
```
1. 用户提供账户ID和初始余额
2. 系统验证输入
3. 系统创建账户记录
4. 系统返回账户信息
5. 用户可以立即使用账户
```
