## 🎯 系统概述

金币交易协议（Coin Exchange Protocol）是一个高性能、高可用的数字资产交易平台，支持多种加密货币的交易、转账和管理。

### 核心特性

- ⚡ **高性能**: 支持 10,000+ TPS 交易处理
- 🔒 **高安全**: 多层安全防护，冷热钱包分离
- 🌐 **高可用**: 99.99% SLA 保证，多地域部署
- 🔧 **易扩展**: 微服务架构，水平扩展能力强
- 📊 **实时性**: WebSocket 推送，毫秒级延迟

## 🧩 核心模块

### 1. 账户服务 (Account Service)

**职责**: 用户账户管理、资产余额、KYC 认证

```
Account Service
├── 用户管理模块
│   ├── 注册/登录
│   ├── 密码管理
│   └── 多因素认证 (2FA)
├── 资产管理模块
│   ├── 余额查询
│   ├── 冻结/解冻
│   └── 转账记录
└── KYC 认证模块
    ├── 身份验证
    ├── 文档审核
    └── 风险评估
```

**API 接口**:
- `GET /account/info` - 获取账户信息
- `GET /account/balance` - 查询余额
- `POST /account/transfer` - 内部转账

---

### 2. 交易服务 (Trading Service)

**职责**: 订单管理、撮合引擎、成交记录

```
Trading Service
├── 订单管理模块
│   ├── 创建订单
│   ├── 取消订单
│   └── 查询订单
├── 撮合引擎模块
│   ├── 价格优先
│   ├── 时间优先
│   └── 成交匹配
└── 成交记录模块
    ├── 成交历史
    ├── 资金清算
    └── 手续费计算
```

**撮合算法流程**:

```
1. 接收新订单
   ↓
2. 验证订单有效性
   ↓
3. 查询对手盘订单
   ↓
4. 价格匹配检查
   ├─ 买单: price >= 最优卖价
   └─ 卖单: price <= 最优买价
   ↓
5. 生成成交记录
   ↓
6. 更新订单状态
   ↓
7. 发送成交通知
```

**API 接口**:
- `POST /trade/order` - 创建订单
- `DELETE /trade/order/{id}` - 取消订单
- `GET /trade/open-orders` - 查询当前订单
- `GET /trade/history` - 查询历史订单

---

### 3. 行情服务 (Market Service)

**职责**: 市场数据推送、K线生成、深度维护

```
Market Service
├── 行情推送模块
│   ├── 实时价格
│   ├── 24h 涨跌
│   └── 成交量统计
├── K线数据模块
│   ├── 多周期生成
│   ├── OHLCV 计算
│   └── 历史查询
├── 深度数据模块
│   ├── 买卖盘维护
│   ├── 聚合计算
│   └── WebSocket 推送
└── 成交记录模块
    ├── 最近成交
    └── 大单监控
```

**WebSocket 频道**:
- `{symbol}@ticker` - 实时行情
- `{symbol}@depth` - 深度数据
- `{symbol}@trade` - 成交记录
- `{symbol}@kline_{interval}` - K线数据

---

### 4. 冻结服务 (Freeze Service)

**职责**: 风控管理、账户/资产冻结

```
Freeze Service
├── 账户冻结模块
│   ├── 全账户冻结
│   ├── 定时解冻
│   └── 冻结记录
├── 资产冻结模块
│   ├── 部分资产冻结
│   ├── 订单锁定
│   └── 风控冻结
└── 风控规则模块
    ├── 异常检测
    ├── 自动冻结
    └── 人工审核
```

**冻结流程**:

```
触发冻结
   ↓
1. 验证权限
   ↓
2. 检查账户状态
   ↓
3. 执行冻结操作
   ├─ 账户冻结: 禁止所有操作
   └─ 资产冻结: 冻结指定资产
   ↓
4. 记录冻结日志
   ↓
5. 发送通知
   ↓
6. 定时任务检查解冻
```

**API 接口**:
- `POST /freeze/account` - 冻结账户
- `POST /freeze/asset` - 冻结资产
- `GET /freeze/records` - 查询冻结记录

## 🔄 数据流程

### 1. 下单流程

```
用户下单请求
    ↓
API Gateway (认证/限流)
    ↓
Trading Service
    ├→ 验证余额 (Account Service)
    ├→ 冻结资金 (Account Service)
    └→ 创建订单
    ↓
撮合引擎
    ├→ 查询对手盘
    ├→ 价格匹配
    └→ 生成成交
    ↓
成交处理
    ├→ 更新余额 (Account Service)
    ├→ 扣除手续费
    └→ 记录成交
    ↓
发送通知
    ├→ WebSocket 推送
    ├→ 邮件/短信
    └→ 事件记录 (Kafka)
```

### 2. 充值流程

```
链上交易检测
    ↓
区块链监控服务
    ├→ 确认交易
    └→ 验证金额
    ↓
入账服务
    ├→ 创建充值记录
    ├→ 增加余额 (Account Service)
    └→ 更新状态
    ↓
发送通知
    ├→ WebSocket 推送
    ├→ 邮件/短信
    └→ 风控检查
```

### 3. 提现流程

```
用户提现请求
    ↓
API Gateway (认证/验证)
    ↓
Wallet Service
    ├→ 验证余额 (Account Service)
    ├→ 冻结资产 (Freeze Service)
    └→ 创建提现记录
    ↓
风控审核
    ├→ 自动审核 (小额度)
    └→ 人工审核 (大额度)
    ↓
区块链交互
    ├→ 构建交易
    ├→ 签名交易
    └→ 广播交易
    ↓
确认到账
    ├→ 更新状态
    └→ 发送通知
```

### 容灾方案

1. **多地域部署**: 美东、美西、亚太三地域
2. **数据库主从**: PostgreSQL 主从同步，自动故障转移
3. **Redis 集群**: 3 主 3 从，哨兵模式
4. **Kafka 集群**: 3 节点集群，数据多副本
5. **备份策略**: 
   - 全量备份: 每天凌晨 2:00
   - 增量备份: 每小时
   - 跨地域备份: 异地容灾

## 🔐 安全架构

### 安全层次

```
┌─────────────────────────────────────────┐
│           应用层安全                      │
│  ├─ API 签名验证                          │
│  ├─ JWT Token 认证                       │
│  └─ 权限控制 (RBAC)                      │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────┼──────────────────────┐
│           传输层安全                      │
│  ├─ HTTPS (TLS 1.3)                      │
│  ├─ HSTS                                 │
│  └─ 证书固定 (Certificate Pinning)       │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────┼──────────────────────┐
│           网络层安全                      │
│  ├─ DDoS 防护 (Cloudflare)               │
│  ├─ WAF (Web Application Firewall)       │
│  └─ IP 白名单                            │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────┼──────────────────────┐
│           数据层安全                      │
│  ├─ 数据加密 (AES-256)                    │
│  ├─ 密钥管理 (KMS)                        │
│  └─ 访问审计                             │
└─────────────────────────────────────────┘
```

### 密钥管理

- **API Key**: SHA-256 哈希存储
- **私钥**: HSM (硬件安全模块) 存储
- **数据库加密**: 透明数据加密 (TDE)
- **密钥轮换**: 90 天自动轮换
