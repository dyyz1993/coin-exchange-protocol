## 📋 1. 输入验证完整性

### 1.1 参数必填性检查
- [ ] 所有必填参数都有明确验证
- [ ] 缺少参数时返回明确的错误消息
- [ ] 错误消息包含缺失的参数名称

### 1.2 参数类型验证
- [ ] 字符串参数验证（非空、长度限制）
- [ ] 数字参数验证（范围、精度、非负）
- [ ] 枚举值验证（只接受预定义值）
- [ ] 日期时间格式验证

### 1.3 业务规则验证
- [ ] 转账金额 > 0
- [ ] 冻结金额 > 0
- [ ] 用户 ID 格式验证
- [ ] 余额充足性检查
- [ ] 账户状态检查（未冻结）

### 1.4 边界条件检查
- [ ] 最大转账金额限制
- [ ] 最小转账金额限制
- [ ] 冻结期限合理性
- [ ] 分页参数边界（page >= 1, limit >= 1）

---

## 🔒 2. 并发安全检查

### 2.1 数据库事务
- [ ] 转账操作使用事务包装
- [ ] 冻结/解冻操作使用事务包装
- [ ] 事务隔离级别正确（READ COMMITTED 或更高）
- [ ] 事务超时设置合理

### 2.2 乐观锁机制
- [ ] 账户余额更新使用版本控制
- [ ] 更新失败时正确重试
- [ ] 重试次数有限制（避免无限循环）

### 2.3 悲观锁机制（如适用）
- [ ] SELECT FOR UPDATE 正确使用
- [ ] 锁的获取顺序一致（避免死锁）
- [ ] 锁持有时间最小化

### 2.4 幂等性保证
- [ ] 转账操作支持幂等性（requestId 去重）
- [ ] 冻结操作支持幂等性
- [ ] 解冻操作支持幂等性

## ⚠️ 3. 错误处理完整性

### 3.1 错误类型覆盖
- [ ] 参数验证错误（400 Bad Request）
- [ ] 认证错误（401 Unauthorized）
- [ ] 权限错误（403 Forbidden）
- [ ] 资源不存在（404 Not Found）
- [ ] 业务逻辑错误（409 Conflict）
- [ ] 服务器内部错误（500 Internal Server Error）

### 3.2 错误消息规范
- [ ] 错误消息清晰易懂
- [ ] 错误消息不泄露敏感信息
- [ ] 错误消息包含错误代码
- [ ] 错误消息支持多语言（如需要）

### 3.3 异常捕获
- [ ] 所有异步操作都有 try-catch
- [ ] 数据库异常正确处理
- [ ] 外部服务异常正确处理
- [ ] 未知异常有默认处理

### 3.4 日志记录
- [ ] 错误日志包含完整上下文
- [ ] 日志级别正确（error/warn/info/debug）
- [ ] 敏感信息不在日志中输出
- [ ] 日志格式统一

## 💾 4. 事务保护验证

### 4.1 事务边界清晰
- [ ] 事务开始和提交/回滚明确

### 4.2 回滚机制
- [ ] 异常时自动回滚
- [ ] 手动回滚条件明确
- [ ] 部分失败时的补偿机制

### 4.3 数据一致性
- [ ] 转账双方余额总和不变
- [ ] 冻结记录与账户状态一致
- [ ] 交易记录完整无遗漏
